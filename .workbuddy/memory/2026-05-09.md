## 项目启动：校园食堂窗口评价系统

### 需求确认
- 系统定位：用户评价食堂窗口菜品，管理员统一后台管理
- 评分对象：菜品（1-5整数星），窗口评分由菜品均分自动计算
- 用户体系：普通用户 + 管理员独立账号，全局用户名唯一
- 登录设计：单一登录页，两个登录框（用户/管理员各一个），后端各自验证
- 图片功能：不支持
- 第6张表：replies（管理员回复评价）

### 数据库（6张表）
1. users - 普通用户
2. admins - 管理员（默认账号：admin/caramel133）
3. windows - 窗口（含color字段，6种预设颜色）
4. dishes - 菜品（is_active上下架）
5. reviews - 评价（UNIQUE user_id+dish_id）
6. replies - 管理员回复评价

### 初始数据
- 6个窗口示例（川菜/粤菜/面食/清真/西式/甜品）
- 每个窗口4道菜品，共24道菜品
- 管理员默认账号：admin / caramel133
